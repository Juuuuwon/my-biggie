Basic APIs

GET /simple

This endpoint responds with a simple JSON message "ok". No parameters required.

GET /simple/foo

Returns "foo ok" along with request header information (cookies, method, IP address, and query parameters).

POST /simple/bar

Sends JSON data and responds with "bar ok" along with header and body details.

GET /simple/color (Not JSON)

Returns an HTML page with a random background color. Override using ?color= (e.g., "#ff0000" or "RANDOM").

Health & Metadata APIs

GET /healthcheck

Simple health check. Returns "ok" in JSON format.

Stress Test APIs

POST /stress/cpu

Maintains the specified cpu_percent for maintain_second seconds. If async is true, returns immediately while the stress test runs in the background. Example default: {"cpu_percent": "RANDOM:10:50", "maintain_second": 30, "async": true}

POST /stress/memory

Maintains the specified memory_percent for maintain_second seconds. Example: {"memory_percent": "RANDOM:10:50", "maintain_second": 30, "async": true}

POST /stress/memory_leak

Gradually allocates a specified amount of memory (leak_size_mb) for maintain_second seconds. Example: {"leak_size_mb": 50, "maintain_second": 30, "async": true}

POST /stress/filesystem/write

Writes multiple files of specified file_size per interval. Example: {"file_size": 1024, "file_count": 10, "maintain_second": 30, "async": true, "interval_second": 1}

POST /stress/filesystem/read

Reads from a specified file repeatedly. Example: {"file_path": "/tmp/testfile", "maintain_second": 30, "async": true, "read_frequency": 10, "interval_second": 1}

POST /stress/network/latency

Introduces artificial network latency. Example: {"latency_ms": 200, "maintain_second": 30, "async": true}

POST /stress/network/packet_loss

Simulates network instability by dropping a percentage of packets. Example: {"loss_percentage": 20, "maintain_second": 30, "async": true}

POST /mysql/heavy

Performs heavy queries on a single connection. Executes SELECT 1 for reads and INSERT into biggie_test_table for writes. Example: {"reads": true, "writes": true, "maintain_second": "RANDOM:30:60", "async": true, "query_per_interval": 10, "interval_second": 1}

POST /mysql/multi_heavy

Executes heavy queries using multiple connections. Example: {"reads": true, "writes": true, "maintain_second": 30, "async": true, "connection_counts": 10, "query_per_interval": 10, "interval_second": 1}

POST /mysql/connection

Simulates heavy connection loads by gradually establishing connections. Example: {"maintain_second": 30, "async": true, "connection_counts": 100, "increase_per_interval": 10, "interval_second": 1}

Kafka APIs

Sends messages to a Kafka topic using a single producer connection. Example: {"messages": "", "maintain_second": 30, "async": true, "produce_per_interval": 10, "interval_second": 1}

POST /kafka/multi_heavy

Error Injection APIs

POST /stress/error_injection

Randomly injects errors into API responses at a defined rate. Example: {"error_rate": "RANDOM:0.05:0.15", "maintain_second": 30, "async": true}

POST /stress/crash

Simulates an unexpected service crash after a brief delay. Example: {"maintain_second": 10, "async": true}

Concurrency & DDoS APIs

POST /stress/concurrent_flood

Simulates a flood of concurrent requests to a target endpoint. Example: {"target_endpoint": "/simple", "request_count": 1000, "maintain_second": 30, "async": true, "interval_second": 1}

POST /stress/downtime

Temporarily disables responses to simulate downtime. Example: {"downtime_second": 10, "async": true}

POST /stress/third_party

Simulates calls to an external API. Example: {"target_url": "https://api.example.com/data", "maintain_second": 30, "async": true, "call_rate": 10, "interval_second": 1, "simulate_errors": true}

POST /stress/ddos

Simulates a DDoS attack by sending high-volume requests. Example: {"target_endpoint": "/simple", "attack_intensity": 1000, "maintain_second": 30, "async": true, "interval_second": 1}

Fake Log Generation API

POST /stress/logs

Generates log messages over time with random content. The logs are produced using a log format defined by the LOG_FORMAT environment variable.
Parameters:

  • maintain_second: Total duration (in seconds) during which logs are generated.
  • log_count_per_interval: Number of log messages generated each interval. Supports RANDOM syntax (e.g., "RANDOM:5:15").
  • line_per_log: Number of lines in each generated log message.
  • interval_seconds: Time (in seconds) between each log generation cycle.
  • async: If true, the API returns immediately while log generation continues in the background.

Example: {"maintain_second": 30, "log_count_per_interval": "RANDOM:5:15", "line_per_log": 3, "interval_seconds": 1, "async": true}
